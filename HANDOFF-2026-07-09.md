## 5. Sandbox environment hazards (CRITICAL — this cost hours)

- **Host↔mount sync corruption**: mount serves stale/truncated views (new content clamped at old byte length; single chars garbled, e.g. `//` → `\ `). NEVER trust Edit/Write tool output alone for this repo, and never "repair" files from mount views.
- **Reliable write path**: base from `git show HEAD:<path>` (or a python read that asserts every replacement count == 1), apply exact replacements in python, write via bash to `/sessions/<session>/mnt/ai-labs-portfolio/...` (write-through to host), then verify bytes + sha + tail + zero NULs.
- **Host `.git/index` is locked/stale**: commit with `export GIT_INDEX_FILE=/tmp/gidxN && git read-tree HEAD && git add <files> && git -c user.name="Sudeep Lalka" -c user.email="sudeeplalka@gmail.com" commit`. Verify with `git cat-file -s HEAD:<path>` == written size.
- **Gates recipe** (sandbox can't run `next build`; Vercel is the authoritative build):
  - `/tmp/repo` = tar copy of the mount (excludes .git; node_modules comes along, 646MB, works).
  - tsc: `cd /tmp/repo/apps/web && ../../node_modules/.bin/tsc --noEmit`
  - eslint: `cd /tmp/repo/apps/web && ../../node_modules/.bin/eslint --no-eslintrc -c .eslintrc.json --resolve-plugins-relative-to ../.. <files>`
  - vitest: `cd /tmp/repo/packages/<pkg> && ./node_modules/.bin/vitest run`
  - Copy changed files into /tmp/repo before gating. VM recycle wipes /tmp → re-tar.
- 45s bash timeout per call; no cwd/env carryover; pnpm not on PATH (use node_modules/.bin directly).
