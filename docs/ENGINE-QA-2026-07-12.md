## 3. Defects found and fixed

**D1 — lab-governance test suites never ran (harness missing).** `governance.test.ts` (16 tests: injection→BLOCK, PII→REDACT, decision precedence, risk simulation) and `demo-data.test.ts` (3) import via the `@gov/*` alias, but the package had no vitest config, no vitest devDependency, and no test script — `turbo test` skipped it silently since the suites were written. Fix: `vitest.config.ts` (test-only `@gov` alias, same pattern as lab-data), vitest 2.0.5 devDep + scripts, lockfile updated (verified with a frozen-lockfile install: "Lockfile is up to date"). First-ever execution: **19/19 green** — the engine itself was correct all along. `components/__tests__/Badge.test.tsx` stays excluded with a comment: it needs `@testing-library/react` + jsdom, a harness this package doesn't ship; add deliberately or delete the orphan test.

**D2 — typecheck gap from Phase 0 closed.** Phase 0 (see `COLLECTION5-LIVE-BUILDS-PLAN.md` §Phase 0) found lab-data's test files were reached by no compiler and left the same gap open in lab-rag and lab-governance. Both now have a package `tsconfig.json` + `typecheck` script. Both compile clean on first run (0 errors) — unlike lab-data, which had 20 latent errors when its gate was added.

**D3 — live evaluator penalized verbatim quotes (scoring defect).** `textUtils.tokenize` kept sentence-final periods inside tokens, so an answer quoting the evidence exactly ("Receipts needed above $25.") produced the token `$25.`, which failed to match the context token `$25` and was scored *ungrounded* — depressing faithfulness (~91 for a fully grounded answer) and inflating hallucination risk. The BM25 retriever already strips punctuation from its terms; evaluation now normalizes the same way (trailing `.`/`-` stripped; internal dots like `$2.5` and trailing `%` survive). A fully grounded, cited answer now scores faithfulness ≥ 95 and gate "Passed". All 38 lab-rag tests green after the change.

## 4. Wiring audit (clean)

- **engines → UI:** consumed through facades as designed — `runEvalBench`/`confusionAt`/`thresholdEconomics` (EvalBench, LB-03), `forecastRunRate`/`cliffSensitivity` (InferenceForecaster), finance fns (portfolio/ROI). Exports unused by the app (`rocCurve`, `genFraudDataset`, …) are **not dead code**: they are the public API of the publish-prepped `ai-decision-engines` npm package, exercised by its 161 tests.
- **Pricing single-source:** `kit/pricing.ts` is the only per-model price table (consumer: CostSimulator). `engines/cost.ts` defines shape + math only. InferenceForecaster's `CHEAP_PRICE`/`FRONTIER_PRICE` are deliberately *blended* $/1M parameters, disclosed in the on-page formula copy — parametrization, not a duplicate table.
- **BM25 twice, by design, now locked:** lab-data's proof engine and lab-rag's retriever implement Okapi BM25 independently (different tokenization layers, one has a disclosed topic bonus, the other meta-word/expansion weighting). Identical constants (k1=1.5, b=0.75) and IDF shape. `bm25-parity.test.ts` pins the raw formula numerically (hand-computed 0.990210) in lab-data; `retrieval.test.ts` locks rank order, TF saturation, and length normalization in lab-rag. Drift in either now fails its own suite; each file cross-references the other.
- **Evaluator chain:** `evaluateLiveAnswer` → LiveLabView; `scoring.ts` → 6 dashboard components; `getRetriever` → live lab. The embedding retriever is an honest placeholder (delegates to BM25, documented) — now locked by a delegation test so it can't silently diverge before a real implementation lands.

## 5. Enhancement candidates (not built)

1. **Real embedding retrieval** behind `getRetriever("embedding")` — the seam is ready; BYO-key transformers.js or provider embeddings would earn the live lab a semantic mode.
2. **Badge.test.tsx harness decision** — add `@testing-library/react` + jsdom to lab-governance, or delete the orphan test (it is the only component test in any package).
3. **Evaluator calibration curve** — the invariants are locked; a small golden set of (question, answer, expected-gate) triples from the sample corpus would catch semantic regressions numbers alone miss.
4. **`engines` internal-only helpers** (`csvCell`, `toCsv`, `diffNames`) — consider `@internal` doc tags or a separate entrypoint before the npm publish, so the public API is the deliberate one.
